여러분은 iOS에서 부드러운 스크롤에 익숙하실 것입니다. 이것을 제대로 구현하려면 코드를 많이 써야 되는데요. Framer의 ScrollComponent는 이것을 완벽하게 대응하고, 커스터마이징도 가능하게 합니다.

ScrollComponent는 두 개의 레이어로 구성됩니다. 첫 번째로 ScrollComponent 자체는 마스킹 레이어로 사용됩니다.

둘째, draggable이 활성화 돼 있고 드래그 영역이 제한되어있는 콘텐츠 레이어입니다. 콘텐츠 레이어의 크기는 자녀 콘텐츠의 크기에 따라 계산됩니다.



# ScrollComponent 만들기
scroll = new ScrollComponent
    size: 120
 
# 콘텐츠레이어 만들기
layerA = new Layer
    parent: scroll.content
 
layerB = new Layer
    parent: scroll.content

드래그 가능한 레이어와 비슷하게, 여러분은 스크롤 방향을 통제할 수 있습니다.


scroll.scrollHorizontal = true
scroll.scrollVertical = false

감싸기

프레이머로 디자인을 동적으로 완성하는 가장 좋은 방법은 가져오기 기능을 사용하는 것입니다. 가져온 레이어에 스크롤을 추가하려면 ScrollComponent.wrap()을 사용하여 ScrollComponent로 감싸주면 됩니다.


# 스케치 파일 불러오기
sketch = Framer.Importer.load("imported/scrollable@1x")
 
# 가져온 콘텐츠레이어를 ScrollComponent로 감싸주기
scroll = ScrollComponent.wrap(sketch.content)

콘텐츠 삽입

콘텐츠에 추가 간격을 추가하려면 contentInset 속성을 사용할 수 있습니다. 이는 스크롤 영역 내에 내용을 배치할 때 유용합니다. 목록 상단의 헤더를 생각해보십시오. 헤더 부분은 스크롤 할 수는 없지만, 콘텐츠 부분은 스크롤 할 수 있습니다. 이것의 예는 iOS용 트위터에서 볼 수 있습니다.



# ScrollComponent 만들기
scroll = new ScrollComponent
    width: 120
    height: 120
    scrollHorizontal: false
 
#contentInset 정의하기
scroll.contentInset =
    top: 40
    bottom: 40
    right: 0
    left: 0

이벤트

세 가지 기본 스크롤 이벤트는 ScrollStart, Scroll (또는 ScrollMove) 및 ScrollEnd가 있습니다. Scroll 이벤트 내에서 스크롤 가능한 레이어의 위치를 검색할 수 있습니다. 예를 들어, scroll.scrollY를 사용하여 세로 스크롤링 거리를 기준으로 애니메이션을 시작할 수 있습니다.



# ScrollComponent 만들기
scroll = new ScrollComponent
    scrollHorizontal: false
 
# Scroll 이벤트 정의하기 
scroll.onScroll ->
    if scroll.scrollY < -10
 
        layerA.animate
            scale: 1

스크롤 애니메이션

드래그 가능한 레이어와 마찬가지로 모멘텀과 바운스가 활성화되어있을 때 발생하는 두 가지 이벤트가 있습니다. ScrollAnimationDidStartand와 ScrollAnimationDidEnd인데요. ScrollEnd 다음에 발생합니다.



# 스크롤이 끝난 뒤, ScrollAnimation이 시작
scroll.onScrollAnimationDidStart ->
    layer.animate
        width: 100
 
# 스크롤 애니메이션 후를 정의하기
scroll.onScrollAnimationDidEnd ->
    layer.animate
        width: 120
